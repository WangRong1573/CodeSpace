final：
    final修饰的类：
        不能被继承
    final修饰的方法：
        不能被重写
    final修饰的变量：
        只能赋值一次，不能重新赋值
    final修饰的引用：
        只能指向一个对象

final修饰的实例变量：
    系统不会赋默认值，需要手动赋值；赋值的方法有两种，一种是在生命的时候直接赋值，一种是在构造方法中赋值；
    因为系统赋默认值是在调用构造方法的时候为实例变量赋值。

    final修饰的实例变量一般添加static修饰，实例变量在堆内存中，一个变量一份，占用内存空间，所以会提取常量。
    示例：static final int i=10；成为常量。常量名全部大写，每个单词之间使用下划线连接。

常量和静态变量区别：
    都是存储在方法区，都是在类加载时初始化；
    常量的值是不变的，名称全部大写，单词之间使用下划线连接。
    常量一般都是公开的。
    public static final String COUNTRY="中国";
